Session

セッション単位の概要情報。JSONLファイル1つにつき1つのSessionが生成される。

フィールド 型 NULL 制約 説明
session_id String NOT NULL PK セッション一意識別子。JSONLファイル名（拡張子除く）から取得
project_path String NOT NULL — プロジェクトのパス。~/.claude/projects/ 配下のディレクトリ名
first_timestamp DateTime NOT NULL — セッション内の最初のメッセージのタイムスタンプ
last_timestamp DateTime NOT NULL — セッション内の最後のメッセージのタイムスタンプ
message_count usize NOT NULL — セッション内のメッセージ総数
cwd String NOT NULL — セッション開始時の作業ディレクトリ

Message

セッション内の個々のメッセージ。user と assistant の両方を保持する。

フィールド 型 NULL 制約 説明
session_id String NOT NULL FK → Session 所属するセッションのID
index usize NOT NULL — セッション内のメッセージ順序（0始まり）
role Role (enum) NOT NULL — User または Assistant
content String NOT NULL — メッセージ本文。content配列の場合はtextフィールドを結合
timestamp DateTime NOT NULL — メッセージのタイムスタンプ（ISO 8601）

SearchIndex（ランタイム）

永続化しないランタイム構造。ファジー検索時にセッションをスコアリングする。

フィールド 型 NULL 制約 説明
query String NOT NULL — ユーザーが入力した検索クエリ
score f64 NOT NULL — ファジーマッチのスコア（高い方が関連度が高い）
matched_fragments Vec<String> OPTIONAL — マッチした箇所のテキスト断片（ハイライト用）

リレーション一覧

親 子 カーディナリティ 外部キー 説明
Session Message 1 : 0..N Message.session_id → Session.session_id 1つのセッションが0個以上のメッセージを持つ
Session SearchIndex 1 : 0..1 ランタイム参照 検索時に各セッションにスコアが付与される

補足: データソース

すべてのデータは ~/.claude/projects/<project-path-hash>/<session-id>.jsonl から読み取り専用でロードされます。
ccc は一切のデータ書き込みを行いません。永続化が必要な機能（ブックマーク等）はPhase 2 で別途検討します。
